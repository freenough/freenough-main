# 指示書: FREENOUGHトップページ 調整(第3ラウンド)

作成日: 2026-07-19
対象: `freenough-main`の`app/page.tsx`
種別: 追加修正
補足: `freenough_top_page_adjustments.md`の完了報告を踏まえたさらなる調整

---

## 1. ページ全体の幅をLifeCompassに近づける(ただし本文の行長は保つ)

現状、ページ全体が`max-w-2xl`のコンテナに収まっているため、LifeCompassのLP(`max-w-7xl`程度)と比べて視覚的に小さく・寂しく見える。

**対応方針**:単純に全部を`max-w-7xl`にはしない。以下のように使い分ける:

- **ページ全体の外枠**:ヘッダーと同じ`max-w-7xl`(または近い値)に広げ、LifeCompassと視覚的なスケール感を揃える
- **Hero・ブランドストーリーの本文テキスト**:読みやすさを保つため、`max-w-2xl`程度のナロー幅のまま、外枠の中央に配置する(1行の文字数が長くなりすぎないようにする)
- **LifeCompassカード**:下記4で幅を広げるため、外枠に近い幅まで許容してよい(本文ほど厳密にナロー幅にこだわらない)

## 2. Heroのフォントサイズを大きくする

- 現状のサイズから1〜2段階アップ(例:`text-5xl sm:text-6xl`程度)
- 2行構成(あなたにとっての「足りる」を、/数字で描く。)は維持する
- サイズアップに伴い、行間・改行位置が崩れないか確認すること(特に「足りる」の括弧書きが不自然な位置で折り返されないか)

## 3. LifeCompassカードを大きく・存在感のある見た目にする

- カード自体の横幅・パディングを広げ、全体のスケール感に合わせて大きくする
- **コンパスロゴ(`compass_logo.png`)を現状より大きく表示する**(目安:現状の倍程度のサイズ感。LifeCompassのLP側で使われているロゴサイズも参考にすること)
- ロゴを大きくすることで、テキスト(「LifeCompass」「あなたの"足りる"を、シミュレーションで確かめる。」)とのバランスが崩れないよう、レイアウト(横並びか、ロゴを上・テキストを下に置くか)は実装時に見た目のバランスを見ながら調整してよい

## 4. フッターを拡充する

現状「© FREENOUGH — Design Your Enough.」の1行のみ。以下の構成に拡充する(お問い合わせは今回対象外):

```
About
X
Note
GitHub

© 2026 FREENOUGH
```

- リンク先URL:
  - About:`https://freenough-lifecompass.vercel.app/about`(LifeCompass側にすでにある「運営者情報」ページをそのまま流用する。新規ページは作らない)
  - X:`https://x.com/freenough`
  - Note:`https://note.com/freenough`
  - GitHub:`https://github.com/freenough`
- 「お問い合わせ」は、現時点で実際の問い合わせ手段(フォーム・メールアドレス)が存在しないため、今回はフッターに含めない(プレースホルダーのダミーリンクは置かない)
- レイアウトは横並び(中央寄せ)で、リンク群の下に著作権表記を配置する形でよい
- 「Design Your Enough.」のタグラインは、著作権表記に含める形(例:「© 2026 FREENOUGH — Design Your Enough.」)のまま残してよい

## 5. その他の細部修正

### 5-1. favicon
- `app/favicon.ico`が`create-next-app`のデフォルトのままになっている。`compass_logo.png`をベースにしたfaviconに差し替える(favicon生成が必要な場合は、既存の`compass_logo.png`から適切なサイズ・形式に変換して配置すること)

### 5-2. OGP画像の設定
- `app/layout.tsx`のmetadataに、OGP画像(`openGraph.images`)が未設定になっている
- シンプルなブランド用OGP画像(1200×630px程度、白背景に「FREENOUGH — Design Your Enough.」のテキストとロゴのみ、LifeCompassの機能訴求要素は含めない)を用意し、`public/images/`に配置してmetadataに設定する
- 画像素材がまだない場合は、その旨を完了報告に明記し、KENZOに確認すること(仮のプレースホルダー画像で進めるか、素材待ちにするか判断を仰ぐ)

### 5-3. 不要なダークモード対応クラスの削除
- 前回の実装で、ブランドストーリー部分の背景色に`bg-slate-50 dark:bg-slate-900/40`のような`dark:`プレフィックス付きクラスが混入している
- LifeCompass側はダークモード対応していないため、Freenough側だけダークモード用のスタイルが部分的に入っていると、ユーザーのOS設定によってはLifeCompassとの配色の一貫性が崩れる
- `dark:`プレフィックスの付いたクラスをすべて洗い出し、意図的なものでなければ削除する(ライトモード固定のスタイルに統一する)

### 5-4. ロゴ画像のalt属性確認
- コンパスロゴ画像(`compass_logo.png`)を表示している箇所すべてに、適切な`alt`属性(例:`alt="LifeCompass"`)が設定されているか確認する。未設定の場合は追加する
