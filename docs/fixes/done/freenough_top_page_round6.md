# 指示書: FREENOUGHトップページ 調整(第6ラウンド)

作成日: 2026-07-19
対象: `freenough-main`の`app/page.tsx`
種別: 追加修正(第4ラウンドの「1画面に収める」方針を撤回し、LifeCompassの余白に揃える)

---

## 背景

第4ラウンドで「デスクトップの一般的な画面高さでフッターまで1画面に収める」ことを目標にセクション間の余白を詰めたが、この目標自体に無理があった。画面の高さはPC・ブラウザ設定によって様々であり、縦長のウィンドウで見ると、詰めた分の反動でLifeCompassカード〜フッターの間に不自然な空白が集中してしまい、かえって間延びした印象になることが実機確認で判明した。

**「1画面に収める」という目標は撤回する**。代わりに、LifeCompass側の実際の余白値(`src/app/page.tsx`で確認済み)にセクション間の余白を揃え、画面の高さに関わらず一定の心地よいリズムで表示されるようにする。

---

## 1. 参考値(LifeCompass側で実際に確認済みの値)

- Heroセクション:`py-16`(上下64px)
- Hero直後の特徴カードセクション:`pt-12`(上48px)
- つまりHero〜次セクションの間隔は実質112px程度

## 2. Freenough側の対応

- Heroセクションのpaddingを、現状の`pt-12 pb-6 sm:pt-14`から、LifeCompass側に近い値(`py-16`程度)に戻す
- ブランドストーリーセクションのpadding-topも、現状の`py-9`から、LifeCompass側の余白感(Hero〜次セクション間が実質112px程度になるよう)に近づける
- 他のセクション(ブランドストーリー→カード、カード→フッター)についても、機械的に均等ではなく、LifeCompass側の「セクションごとにpy-12前後」という感覚に近づける

## 3. フッターの固定挙動を確認・解消

- ページ全体のレイアウトで、`min-h-screen`+`flex`等によりフッターを画面下部に固定する実装になっていないか確認する
- なっている場合は、この固定をやめ、コンテンツの自然な高さに応じてフッターがその直後に続く形にする(画面が高くても、コンテンツ量に応じた自然な余白で終わるようにする)

## 4. 確認方法

- 実装後、複数の異なる画面高さ(例:800px・1000px・1300px相当のウィンドウサイズ)で表示を確認し、どの高さでも「不自然に空白が集中している箇所」がないことを確認する
- 特に、縦長のウィンドウで見た際に、カード〜フッター間に大きな空白が残っていないか重点的に確認すること

---

## 5. 受け入れ基準

- [ ] Hero・各セクションの余白が、LifeCompass側の実際の値(py-16、pt-12等)に近づいている
- [ ] フッターが画面下部に強制的に固定される実装になっていない(コンテンツの自然な高さに応じて配置される)
- [ ] 複数の画面高さで確認し、不自然に集中した空白が発生していない
- [ ] `npm run build`が型エラーなしで通る
