## 修正ルール

### 1. 記事ごとに「初出は正式名称、2回目以降は短縮形」

- 各記事の中で**最初に「LifeCompass」が登場する箇所**(frontmatterのdescription、または本文の最初の言及)は、「**FREENOUGH 資産シミュレーター**」に置き換える
- **同じ記事内の2回目以降の言及**は、「**資産シミュレーター**」(FREENOUGHを省略した短縮形)に置き換える
- 記事が変われば、また最初の言及は正式名称に戻る(記事ごとに独立してカウントする)

### 2. 見出し・リンクテキストは常に短縮形

以下のような定型パターンは、記事内の何回目の言及であっても、常に短縮形「資産シミュレーター」を使う:

- 見出し「## LifeCompassで分かったこと」→「## 資産シミュレーターで分かったこと」
- 見出し「## LifeCompassで検証する」(および「## LifeCompassで検証する:〜」のような複合見出し)→「## 資産シミュレーターで検証する」
- リンクテキスト「LifeCompassで試算する」→「資産シミュレーターで試算する」
- リンクテキスト「LifeCompassで比較する」→「資産シミュレーターで比較する」

### 3. 「シミュレーター」「シミュレーション」の重複回避

「資産シミュレーターで〜をシミュレーションし」のように、同一文中で「シミュレーター」と「シミュレーション」が近接して重複感が出る箇所は、動詞を「試算する」等に言い換えて重複を避ける。

例:
- 変更前:「LifeCompassでインフレ率0〜3.5%をシミュレーションし、資産が尽きる年齢への影響を検証します。」
- 変更後:「資産シミュレーターでインフレ率0〜3.5%を試算し、資産が尽きる年齢への影響を検証します。」

このパターンに該当する箇所は、調査報告の中でも複数確認されている(fire-inflation-sensitivity.mdのdescription等)。1文ずつ確認し、重複が起きる箇所は言い回しを調整すること。

### 4. リンク先URL(`https://freenough-lifecompass.vercel.app/...`)は変更しない

ドメイン名の一部である小文字の`lifecompass`は、今回のサービス名表記とは別物であり、修正対象外(B-1で実装済みの仕組みにより表示時に正規ドメインへ自動変換される)。
